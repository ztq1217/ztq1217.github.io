java 基础（二）

发表于 2020-10-05 | 分类于 java

大二是要学java的，暑期也简单接触过java，但是过了一两个月又忘记了。
然后这几天重新学了下，基于的教程是 how2j.cn

Chap 7 类和对象

引用

如果一个变量的类型是 类 类型，而非基本类型，那么该变量又叫做引用。
new Item() 创建一个对象，但是这个对象访问不到，就像他没有名字一样。
此时需要一个变量代表(指向)这个对象，这就需要Item money = new Item()

一个对象可以有多个引用，就像一个人可以有很多代号，指向的都是同一个对象。
一个引用，同一时间，只能指向一个对象。

继承

设计一个类的时候，可以继承其他类的内容。
假设Item类有name price两个属性
我们可以设计一个weapon类继承`item·

重载

方法的重载指的是方法名一样，但是参数类型不一样

参数数量是可变的，当有很多很多参数时，可以设计采用下面的方法

构造方法

一般来说直接Item money = new Item()就能创建一个对象
此时Item()就是构造方法

可以不带参数

可以带参数

实际应用大概是直接通过参数赋初值。

构造方法也可以重载，有多个参数。

this

this即代表当前对象

通过this访问属性，这样参数名可以跟属性名相同。

如果要在一个构造方法中，调用另一个构造方法，可以使用this()

看起来就像是套娃。

包

把比较接近的类，规划在同一个包下

Hero,ADHero规划在一个包，叫做charactor(角色)
Item,Weapon规划在另一个包下，叫做property(道具)

使用其他包下的类，必须import

访问修饰符

成员变量有四种修饰符
private 私有的
package/friendly/default 不写
protected 受保护的
public 公共的

红色字体，表示不可行

  1. 属性通常使用private封装起来
  2. 方法一般使用public用于被调用
  3. 会被子类继承的方法，通常使用protected
  4. package用的不多，一般新手会用package,因为还不知道有修饰符这个东西

再就是作用范围最小原则
简单说，能用private就用private，不行就放大一级，用package,再不行就用protected，最后用public。 这样就能把数据尽量的封装起来，没有必要露出来的，就不用露出来了

类属性

当一个属性被static修饰的时候，就叫做类属性，又叫做静态属性
当一个属性被声明成类属性，那么所有的对象，都共享一个值

访问类属性有两种方式
对象.类属性　garen.copyright
类.类属性　hero.copyright

类方法

类似于类属性

访问一个对象方法，必须建立在有一个对象的前提的基础上
访问类方法，不需要对象的存在，直接就访问

单例模式

单例模式又叫做 Singleton模式，指的是一个类，在一个JVM里，只有一个实例存在。
public static的 getInstance方法，返回静态属性

枚举类型

枚举enum是一种特殊的类(还是类)，使用枚举可以很方便的定义常量

注：因为是常量，所以一般都是全大写

Chap 8 接口与继承

接口是抽象方法的集合。
接口并不是类，编写接口的方式和类很相似，但是它们属于不同的概念。
类描述对象的属性和方法。接口则包含类要实现的方法。

创建接口

创建一个接口 File->New->Interface

实现接口

实现在语法上使用关键字 implements

接口的类型转换

所有的子类转换为父类，都是说得通的

父类转子类，有的时候行，有的时候不行，所以必须进行强制转换。

没有继承关系的两个类，互相转换，一定会失败

接口重写

子类可以继承父类的对象方法
子类和父类中有同样名字的方法时
调用方法就会执行子类中重写的方法，而不是用父类的方法
